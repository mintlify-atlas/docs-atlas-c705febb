## Overview

The query tools provide read-only access to the user's mastery state:
- **GET /status**: Query mastery for a specific concept or get overview of all concepts
- **GET /zpd-frontier**: Get the Zone of Proximal Development frontier (concepts ready to learn next)

These endpoints are used by:
- Claude Code to check mastery before answering questions
- Dashboard to display progress
- Hooks to decide whether to surface concepts

## Get Status

### Endpoint

```
GET /api/mcp/status?conceptId={id}
```

### Query Parameters

<ParamField query="conceptId" type="string">
  Concept ID to query. If omitted, returns overview of all concepts.
</ParamField>

### Response (Single Concept)

<ResponseField name="concept" type="object">

  <Expandable title="concept object">
    <ResponseField name="mastery" type="number">
      Mastery probability (0-1)
    </ResponseField>
    <ResponseField name="sigma" type="number">
      Posterior standard deviation (uncertainty)
    </ResponseField>
    <ResponseField name="posteriorVariance" type="number">
      Variance (`sigma^2`) — alternative uncertainty measure
    </ResponseField>
    <ResponseField name="confidenceLevel" type="enum">
      Human-readable confidence:
      - `none`: Never assessed
      - `low`: `sigma >= 0.8`
      - `medium`: `0.4 <= sigma < 0.8`
      - `high`: `sigma < 0.4`
    </ResponseField>
    <ResponseField name="assessmentCount" type="integer">
      Total number of assessments (probes + tutor)
    </ResponseField>
    <ResponseField name="lastAssessed" type="string">

    </ResponseField>
    <ResponseField name="tutoredCount" type="integer">
      Number of tutored assessments
    </ResponseField>
    <ResponseField name="untutoredCount" type="integer">
      Number of untutored (organic) assessments
    </ResponseField>
    <ResponseField name="urgency" type="number">
      Probe urgency score (0-1)
    </ResponseField>
  </Expandable>
</ResponseField>


### Query Parameters

<ParamField query="limit" type="integer" default="20">

<ParamField query="includeUnassessed" type="boolean" default="false">
  Include concepts with zero assessments. If `false`, only returns assessed concepts.

## Implementation Details

### Zone of Proximal Development (ZPD)

The ZPD frontier includes concepts where:
1. **Concept is not mastered**: `mastery < 0.7`
2. **All prerequisites are mastered**: For every `requires` edge `C → P`, `P.mastery >= 0.7`

This ensures the user is ready to learn the concept — they have the foundation but haven't mastered it yet.

Implemented via SQL:
```sql
WHERE (1.0 / (1.0 + EXP(-COALESCE(ucs.mu, 0)))) < 0.7
  AND NOT EXISTS (
    SELECT 1 FROM concept_edges ce
    LEFT JOIN user_concept_states pucs ON pucs.concept_id = ce.target_id
    WHERE ce.source_id = c.id
      AND ce.edge_type = 'requires'
      AND (1.0 / (1.0 + EXP(-COALESCE(pucs.mu, 0)))) < 0.7
  )
```

Source: `src/api/routes/mcp.ts:1085-1104`.

### Sorting

Frontier is sorted by:
1. **Assessed first**: Concepts with `assessmentCount > 0` come before unassessed
2. **Fisher info descending**: Within each group, sort by information gain

This prioritizes:
- Concepts we already have some signal on (more targeted probing)
- Concepts where probing will reduce uncertainty the most

Source: `src/api/routes/mcp.ts:1132-1137`.

### Confidence Level

For single-concept queries, `confidenceLevel` is derived from `sigma`:

```typescript
if (assessmentCount === 0) confidenceLevel = 'none';
else if (sigma < 0.4) confidenceLevel = 'high';
else if (sigma < 0.8) confidenceLevel = 'medium';
else confidenceLevel = 'low';
```

Lower sigma = higher confidence in mastery estimate.

Source: `src/api/routes/mcp.ts:1019-1023`.

### Urgency Score

For single-concept queries, urgency is computed via `probeUrgency()` which combines:
- Uncertainty (`sigma`)
- Staleness (days since last assessment + FSRS decay)
- Assessment count (boost novel concepts)
- Fisher information (not used for display, set to 0)

Source: `src/core/probe-urgency.ts` and `src/api/routes/mcp.ts:1035-1041`.
