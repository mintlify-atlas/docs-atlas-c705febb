## Overview

The Mastery API provides access to user concept states based on Item Response Theory (IRT). Key features:

- Retrieve all mastery states for the authenticated user
- Get mastery for individual concepts
- View assessment history
- Explore the Zone of Proximal Development (ZPD) frontier

**Authentication**: All mastery endpoints require authentication.

**Base path**: `/api/mastery`

**Source**: `src/api/routes/mastery.ts`

## Mastery Model

Mastery states use Bayesian IRT with Gaussian priors:

- **μ (mu)**: Mean on IRT scale (real line, typically -3 to +3)
- **σ (sigma)**: Standard deviation (uncertainty, starts at 1.5)
- **Mastery probability**: `p = 1 / (1 + exp(-μ))` (sigmoid transform to 0-1 scale)

**Initial state**: `μ = 0.0, σ = 1.5` (50% mastery, high uncertainty)

1. User's mastery is **below threshold**
2. All prerequisite concepts have mastery **≥ threshold**

**Algorithm**: SQL query with recursive CTE checking prerequisite mastery.

## Converting μ to Mastery Probability

To convert IRT scale (μ) to a 0-1 mastery probability:

```javascript
function pMastery(mu) {
  return 1 / (1 + Math.exp(-mu));
}
```

**Examples**:
- `μ = -2` → `p ≈ 0.12` (12% mastery)
- `μ = 0` → `p = 0.5` (50% mastery)
- `μ = 1` → `p ≈ 0.73` (73% mastery)
- `μ = 2` → `p ≈ 0.88` (88% mastery)

## Rubric Scores

Assessments use a 4-level rubric:

- **0**: Incorrect/No understanding
- **1**: Partial understanding with significant gaps
- **2**: Good understanding with minor gaps
- **3**: Excellent/Complete understanding

**Source**: `src/schemas/types.ts:178`

## Assessment Event Types

- **probe**: Unprompted knowledge check
- **tutor_phase1**: Pre-tutoring assessment
- **tutor_phase4**: Post-tutoring assessment
- **implicit**: Inferred from work product (not explicit Q&A)

**Source**: `src/schemas/types.ts:107`

## Probe Depth Levels

Probe depth follows Bloom's taxonomy:

- **0**: Recall/Recognition
- **1**: Understanding/Comprehension
- **2**: Application
- **3**: Analysis/Evaluation

**Source**: Assessment event schema

## ZPD Algorithm Details

The ZPD frontier query:

1. Converts all μ values to mastery probabilities using sigmoid
2. Filters concepts with mastery < threshold
3. Excludes concepts with any prerequisite below threshold
4. Orders by concept ID

**SQL**: Uses recursive CTE to traverse "requires" edges.
