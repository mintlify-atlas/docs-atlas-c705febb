## Overview

The Entendi API supports three authentication methods:

1. **Session cookies** - For browser-based requests after signing in
2. **Bearer tokens** - For programmatic access with short-lived tokens
3. **API keys** - For long-lived programmatic access

All authenticated endpoints return `401 Unauthorized` if no valid credentials are provided.

## Better Auth Integration

Entendi uses [Better Auth](https://better-auth.com) for authentication, providing:

- Email/password authentication with optional email verification
- Social OAuth providers (GitHub, Google)
- Organization management with role-based permissions
- API key generation for programmatic access
- Device code flow for CLI tools

### Auth Configuration

Better Auth is configured with:

- **Base path**: `/api/auth`
- **Session expiry**: 7 days
- **Session refresh**: After 1 day of inactivity
- **Plugins**: Organization, API Key, Bearer Token, Admin

## Authentication Methods

### 1. Session Cookies

After signing in through the dashboard, your browser automatically includes session cookies with API requests.

```bash
curl https://api.entendi.dev/api/me \
  -H "Cookie: better-auth.session_token=..." \
  -H "Content-Type: application/json"
```

**Sign in endpoints**:
- `POST /api/auth/sign-in/email` - Email/password sign-in
- `POST /api/auth/sign-up/email` - Create account
- `GET /api/auth/signin/github` - GitHub OAuth
- `GET /api/auth/signin/google` - Google OAuth

### 2. Bearer Tokens

Short-lived tokens for programmatic access. Include the token in the `Authorization` header:

```bash
curl https://api.entendi.dev/api/mastery \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json"
```

**Token query parameter**: For SSE/EventSource requests that can't set headers, use the `token` query parameter:

```javascript
const eventSource = new EventSource(
  'https://api.entendi.dev/api/events/stream?token=<token>'
);
```

**Source**: `src/api/index.ts:186-190`

### 3. API Keys

Long-lived keys for CLI tools and integrations. Include the API key in the `x-api-key` header:

```bash
curl https://api.entendi.dev/api/concepts \
  -H "x-api-key: <your-api-key>" \
  -H "Content-Type: application/json"
```

**Generate an API key**:

```bash
curl -X POST https://api.entendi.dev/api/auth/api-key/generate \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My CLI Tool",
    "expiresIn": "30d"
  }'
```

**Rate limiting**: API keys have rate limiting disabled by default (configurable per environment).

**Source**: `src/api/lib/auth.ts:151-155`

## Session Resolution Flow

The API resolves authentication in the following order:

1. Check for `token` query parameter (for SSE)
2. Extract credentials from headers:
   - `x-api-key` header → API key lookup
   - `Authorization: Bearer <token>` → Bearer token validation
   - `Cookie: better-auth.session_token=...` → Session cookie validation
3. Set `user` and `session` context variables
4. Return `429` if API key rate limit exceeded
5. Return `401` on protected endpoints if no valid credentials

**Source**: `src/api/index.ts:184-215`

## Error Responses

### 401 Unauthorized

Returned when authentication credentials are missing or invalid:

```json
{
  "error": "Unauthorized"
}
```

### 429 Rate Limit Exceeded

Returned when API key rate limit is exceeded:

```json
{
  "error": "API key rate limit exceeded"
}
```

### 403 Forbidden

Returned when authenticated user lacks required permissions:

```json
{
  "error": "Forbidden"
}
```

## Rate Limiting

- **API routes**: 200 requests per minute per user/IP
- **Window**: 60 seconds
- **API keys**: Rate limiting disabled by default

**Source**: `src/api/index.ts:218`

## CORS Configuration

The API allows requests from:

- `https://entendi.dev`
- `https://api.entendi.dev`
- `http://localhost:3456` (development)
- Custom origins via `ENTENDI_CORS_ORIGINS` environment variable

**Allowed headers**: `Content-Type`, `Authorization`, `x-api-key`

**Allowed methods**: `GET`, `POST`, `PUT`, `DELETE`, `OPTIONS`

**Source**: `src/api/index.ts:157-172`

Some endpoints require admin privileges:
