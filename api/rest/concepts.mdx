## Overview

The Concepts API manages the knowledge graph of learnable concepts. Concepts represent discrete knowledge units that can be:

- Searched by domain or keyword
- Created individually or in batches
- Updated with metadata (domain, specificity, aliases)
- Linked with prerequisite/hierarchical relationships

**Base path**: `/api/concepts`

**Source**: `src/api/routes/concepts.ts`

## Concept Model

Concepts use normalized identifiers (lowercase, hyphen-separated) and track:

- **Domain**: Broader category (e.g., "react", "typescript", "algorithms")
- **Specificity**: Level of granularity ("domain", "topic", "technique")
- **Aliases**: Alternative names for search/matching
- **Relationships**: Graph edges (requires, part_of, related_to, etc.)

**Source**: `src/schemas/types.ts:37-44`

### Create or Upsert Concept

Create a new concept or update an existing one. IDs are normalized automatically.

<CodeGroup>
```bash cURL
curl -X POST https://api.entendi.dev/api/concepts \
  -H "x-api-key: <your-api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "React Hooks",
    "domain": "react",
    "specificity": "topic",
    "aliases": ["hooks", "useState", "useEffect"],
    "parentId": "react"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.entendi.dev/api/concepts', {
  method: 'POST',
  headers: {
    'x-api-key': '<your-api-key>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    id: 'React Hooks',
    domain: 'react',
    specificity: 'topic',
    aliases: ['hooks', 'useState', 'useEffect'],
    parentId: 'react'
  })
});
const concept = await response.json();
```
</CodeGroup>

#### Request Body

<ParamField body="id" type="string" required>
  Concept identifier (will be normalized to lowercase-hyphenated)
</ParamField>

<ParamField body="domain" type="string">
  Domain category (default: "general")
</ParamField>

<ParamField body="specificity" type="string">
  Granularity: "domain", "topic", "technique" (default: "topic")
</ParamField>

<ParamField body="aliases" type="string[]">
  Alternative names for search/matching
</ParamField>

<ParamField body="parentId" type="string">
  Parent concept ID for hierarchical relationships
</ParamField>

#### Response

Returns the created/updated concept object.

**Status codes**:
- `200 OK` - Concept updated (existing)
- `201 Created` - New concept created
- `400 Bad Request` - Missing or invalid `id`

**Normalization**: Input `"React Hooks"` → stored as `"react-hooks"`

**Upsert behavior**: If concept exists, merges aliases and updates metadata (domain/specificity only if provided).

**Authentication**: Required (`requireAuth`)

**Source**: `src/api/routes/concepts.ts:35-77`

## Concept Identifier Normalization

All concept IDs are normalized on creation/upsert:

1. Convert to lowercase
2. Replace spaces and slashes with hyphens
3. Remove invalid characters (keep only `a-z0-9-_.@`)

**Examples**:
- `"React Hooks"` → `"react-hooks"`
- `"TypeScript/Generics"` → `"typescript-generics"`
- `"AWS S3"` → `"aws-s3"`

**Source**: `src/api/routes/concepts.ts:48-49`

## Edge Types

Supported relationship types:

- **requires**: Prerequisite relationship (A requires B)
- **part_of**: Hierarchical containment (A is part of B)
- **related_to**: Associative relationship
- **alternative_to**: Alternative approaches/techniques
- **used_with**: Commonly used together
- **is_example_of**: Exemplar relationship
