## Prerequisites

Before you begin, make sure you have:

- Claude Code CLI installed and configured
- Node.js 22+ (check with `node --version`)
- A terminal with internet access

<Note>
  Don't have Claude Code? Visit [claude.com/claude-code](https://claude.com/claude-code) to get started.
</Note>

## Installation

<Steps>
  <Step title="Add the plugin from marketplace">
    Add Entendi from the Claude Code plugin marketplace:

    ```bash
    claude plugin marketplace add https://github.com/LemonThyme/entendi
    ```

    Then install it:

    ```bash
    claude plugin install entendi
    ```

    <Tip>
      The plugin installs to your Claude Code plugins directory and registers the MCP server automatically.
    </Tip>
  </Step>

  <Step title="Link your account">
    Start any Claude Code session and run:

    ```
    You: "entendi login"
    ```

    A browser window will open. Sign in (or create a new account) and click **Confirm Link**.

    <Warning>
      Keep the Claude Code session open until you see the confirmation message.
    </Warning>

    After confirming in the browser, tell Claude you're done:

    ```
    You: "done"
    ```

    Your API key is saved to `~/.entendi/config.json` automatically.
  </Step>

  <Step title="Restart Claude Code">
    Exit your current session and start a new one:

    ```bash
    claude
    ```

    Entendi activates automatically on every session after this.
  </Step>
</Steps>

## Verify Installation

Check that Entendi is running correctly:

<AccordionGroup>
  <Accordion title="Option 1: Check plugin status" icon="plug">
    List installed plugins:

    ```bash
    claude plugin list
    ```

    You should see `entendi` in the list.
  </Accordion>

  <Accordion title="Option 2: Test concept detection" icon="vial">
    Start a Claude Code session and ask for something technical:

    ```
    You: "Explain how V8 isolates work in Cloudflare Workers"
    ```

    After Claude responds, you may receive a probe question like:

    ```
    Claude: "Quick check — why can't you use fs or child_process in Workers?"
    ```

    This confirms Entendi is detecting concepts and probing your understanding.
  </Accordion>

  <Accordion title="Option 3: Check debug logs" icon="file-lines">
    View the debug log:

    ```bash
    tail -f ~/.entendi/debug.log
    ```

    You should see entries like:

    ```
    [2026-03-03T10:30:00.000Z] [mcp] tool:entendi_observe called
    [2026-03-03T10:30:01.000Z] [mcp] tool:entendi_observe result
    ```
  </Accordion>
</AccordionGroup>

## Your First Probe

Here's what to expect during your first interaction:

<Steps>
  <Step title="Work normally">
    Continue using Claude Code as you usually do. Ask questions, write code, deploy services.

    ```
    You: "Set up a Postgres connection with Drizzle ORM"
    ```
  </Step>

  <Step title="Claude completes your request">
    Claude will finish your task first:

    ```
    Claude: [generates the code]
    ```

    <Tip>
      Entendi never interrupts work in progress.
    </Tip>
  </Step>

  <Step title="Receive a probe">
    After completing the task, Claude may ask a comprehension question:

    ```
    Claude: "By the way, what's the main benefit of using an ORM 
             like Drizzle instead of raw SQL queries?"
    ```
  </Step>

  <Step title="Respond naturally">
    Answer in your own words:

    ```
    You: "It provides type safety and prevents SQL injection?"
    ```

    Your response is scored on a 0-3 scale:
    - **0**: No understanding
    - **1**: Partial understanding
    - **2**: Good understanding
    - **3**: Complete mastery
  </Step>

  <Step title="Get tutoring if needed">
    If your score is low, Claude offers a Socratic tutor session:

    ```
    Claude: "You're on the right track with SQL injection prevention.
             Let's explore the type safety aspect. What does it mean
             for a database query to be 'type-safe'?"
    ```

    The tutor guides you through understanding without giving direct answers.
  </Step>
</Steps>

## Configuration

Entendi stores configuration at `~/.entendi/config.json`:

```json
{
  "apiKey": "ent_...",
  "apiUrl": "https://api.entendi.dev"
}
```

<Warning>
  Never commit this file to version control. Your API key provides full access to your account.
</Warning>

### Rate Limits

To prevent interruptions, Entendi has built-in rate limits:

- **Minimum probe interval**: 2 minutes between probes
- **Maximum probes per session**: 15 probes per hour

These limits ensure comprehension checking doesn't disrupt your flow.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Login fails with 'device code expired'" icon="clock">
    Device codes expire after 15 minutes. Start a new login:

    ```
    You: "entendi login"
    ```

    Complete the browser confirmation within 15 minutes.
  </Accordion>

  <Accordion title="No probes are appearing" icon="question">
    Check these common issues:

    1. **Not authenticated**: Run `entendi login` to link your account.
    2. **Rate limited**: Wait 2+ minutes between probes.
    3. **Concepts already mastered**: Entendi only probes concepts with uncertainty.

    View debug logs for details:

    ```bash
    tail -f ~/.entendi/debug.log
    ```
  </Accordion>

  <Accordion title="API errors (401/403)" icon="lock">
    Your session may have expired. Re-authenticate:

    ```
    You: "entendi login"
    ```

    This generates a new API key.
  </Accordion>

  <Accordion title="Plugin not loading" icon="plug-circle-xmark">
    Reinstall the plugin:

    ```bash
    claude plugin uninstall entendi
    claude plugin install entendi
    claude
    ```

    Check `~/.entendi/debug.log` for startup errors.
  </Accordion>
</AccordionGroup>
